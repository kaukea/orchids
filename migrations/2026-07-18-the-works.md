# 2026-07-18 — the uncommittable channel is namespaced .git/the-works/

`HANDOVER.md` and `MOOD.md` move from flat in `$(git rev-parse
--git-common-dir)/` into `$(git rev-parse --git-common-dir)/the-works/` —
same guarantees (physically uncommittable, worktree-shared), now with an
identity that cannot collide with git's own files or other tooling. The
migration watermark also lives there (`the-works/migrated`).

## Detect → convert

Mechanical sweep (safe to run as-is; every action is state-guarded):

```sh
gcd="$(git rev-parse --git-common-dir)"
mkdir -p "$gcd/the-works"
[ -f "$gcd/MOOD.md" ] && mv -n "$gcd/MOOD.md" "$gcd/the-works/MOOD.md"
[ -f "$gcd/HANDOVER.md" ] && mv -n "$gcd/HANDOVER.md" "$gcd/the-works/HANDOVER-legacy-flat.md"
root="$(git rev-parse --show-toplevel)"
[ -f "$root/HANDOVER.md" ] && mv -n "$root/HANDOVER.md" "$gcd/the-works/HANDOVER-legacy-root.md"
git worktree list --porcelain | sed -n 's/^worktree //p' | while read -r wt; do
  [ "$wt" = "$root" ] && continue
  name="$(basename "$wt")"
  [ -f "$wt/HANDOVER.md" ] && mv -n "$wt/HANDOVER.md" "$gcd/the-works/HANDOVER-legacy-$name.md"
done
true
```

Provenance-stamped names mean several strays never overwrite each other or
a fresh handover; `mv -n` never clobbers.

## Then: ingest

Any `HANDOVER*.md` now gathered in `.git/the-works/` is ingested per the
`handover` skill's batch rule — all in one pass, oldest-first. Legacy
strays may predate the durable-facts discipline: promote anything
durable-looking (a decision, an outcome, deferred work) to its proper home
during triage instead of assuming it is already there.

## Verify

`.git/the-works/` exists; nothing named `HANDOVER*.md` or `MOOD.md`
remains flat in `.git/` or at any repo/worktree root.
